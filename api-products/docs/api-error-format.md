# Формат ошибок API

`GET /api/products` возвращает ошибки валидации в фиксированном JSON-формате на русском языке.

## Структура

```json
{
    "message": "Ошибка валидации. Исправьте параметры запроса из поля errors и повторите запрос.",
    "errors": {
        "per_page": [
            {
                "message": "Параметр per_page не должен быть больше 100.",
                "suggestion": "Передайте per_page=50 или другое целое число от 1 до 100."
            }
        ]
    }
}
```

## Поддерживаемые параметры

| Параметр      | Возможная ошибка                           | Как исправить                                                        |
|---------------|--------------------------------------------|----------------------------------------------------------------------|
| `page`        | меньше 1 или не целое число                | передать положительное целое число, например `page=1`                |
| `per_page`    | меньше 1, больше 100 или не целое число    | передать `per_page=50` или другое значение от 1 до 100               |
| `category_id` | категория не найдена или значение не целое | передать id существующей категории третьего уровня или убрать фильтр |
| `price_min`   | не число или меньше 0                      | передать неотрицательное число, например `price_min=100.00`          |
| `price_max`   | не число, меньше 0 или меньше `price_min`  | передать корректную верхнюю границу цены                             |
| `in_stock`    | не булево значение                         | передать `true`, `false`, `1` или `0`                                |

## Примечания

- `in_stock=true` и `in_stock=false` нормализуются до boolean перед валидацией.
- Каждая ошибка уровня поля содержит понятное сообщение `message` и практическую подсказку `suggestion`.
