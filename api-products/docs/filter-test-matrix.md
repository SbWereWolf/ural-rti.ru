# Матрица тестов фильтрации каталога

Endpoint: `GET /api/products`.

Матрица покрывает три измерения фильтрации:

| Измерение | Состояния                                                   | Количество |
|-----------|-------------------------------------------------------------|-----------:|
| Категория | без категории, leaf-категория, non-leaf-категория           |          3 |
| Остаток   | любой остаток, `in_stock=true`                              |          2 |
| Цена      | без цены, `price_min`, `price_max`, `price_min + price_max` |          4 |

Итого:

```text
3 × 2 × 4 = 24 сценария
```

`in_stock=false` не включается в матрицу, потому что это не эффективный фильтр. Для него есть отдельный тест:
`in_stock=false` должен вести себя так же, как отсутствие фильтра по остатку.

## Проверки в каждом сценарии

Каждый сценарий проверяет:

```text
HTTP 200
ожидаемый meta.total
ожидаемый meta.per_page
структуру JSON-ответа
соответствие фильтру category_id, если он задан
соответствие фильтру in_stock, если он задан
соответствие фильтрам price_min/price_max, если они заданы
порядок category_id ASC, product_id ASC
формат product.price как строки с двумя знаками после запятой
```
