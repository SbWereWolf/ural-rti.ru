
# Часть 1. Laravel, примерно 20 минут

Нужно сделать REST API эндпоинт `GET /api/products`, который отдаёт
каталог товаров.

## Что должно быть учтено

- Постраничная выдача и фильтрация по категории, диапазону цены и
  наличию на складе.
- В каталоге около миллиона записей, поэтому запрос должен быть
  экономным, подумайте про индексы.
- Можно использовать Eloquent или Query Builder на ваш выбор, коротко
  поясните, почему выбрали этот вариант.
- Не должно быть проблемы N+1 при выдаче связанных данных, например
  категории и остатка.
- Базовая валидация входных параметров и понятная обработка ошибок.

Достаточно прислать миграции или описание структуры таблиц, модель или
модели и контроллер либо сервис. Тесты по желанию, если сделаете, это
будет плюсом.
